Webbläsarspecifik stilsättning

Utvecklare bakom webbläsare kan implementera nya experimentella stilsättningsinställningar för CSS i sina webbläsare i en testperiod där stilsättningen kanske ännu inte stöds bland andra webbläsare. För att använda dessa experimentella stilsättningar används så kallade prefix framför CSS-egenskapen. CSS prefix som används är:

  • -webkit- (Android, Chrome, Safari, iOS)
  • -moz- (Firefox)
  • -ms- (Internet Explorer)
  • -o- (Opera)

Vissa webbläsare förstår alltså inte vissa experimentella CSS-egenskaper utan prefix och det kan därför behövas läggas till prefix framför egenskapen för att den ska kunna fungera. Att skriva samma egenskap flera gånger fast med olika prefix framför är något som bara är temporärt och kommer att ersättas med egenskapen utan prefix när webbläsaren förbättras och får mer stöd för nya CSS-egenskaper.

Exempel 1

För ett par år sedan så behövde man skriva:


.example {
    -moz-border-radius: 10px 5px;
    -webkit-border-top-left-radius: 10px;
    -webkit-border-top-right-radius: 5px;
    -webkit-border-bottom-right-radius: 10px;
    -webkit-border-bottom-left-radius: 5px;
    border-radius: 10px 5px;
}

Webbläsarna har idag förbättrat sitt stöd för CSS-egenskapen och man behöver idag bara skriva:


.example {
    border-radius: 10px 5px;
}

Exempel 2

Den här koden roterar ett element 5 grader.


.example {
    -webkit-transform: rotate(5deg); /* Ch <36, Saf 5.1+, iOS, An =<4.4.4 */
    -ms-transform: rotate(5deg); /* IE 9 */
    transform: rotate(5deg); /* IE 10, Fx 16+, Op 12.1+ */
}
